## Actions Taken

1. **Branched `dev-680` off `main`** (after DEV-679 merged as PR #78). `main` was at `2b891ff`.
2. **Created theme catalog** at `src/lib/themes.ts`:
   - `Theme` / `ThemeTokens` / `ThemeMode` / `ThemeId` types
   - 11 themes (8 light + 3 soft-dark), each with full 19-token palette + 4-color preview chip
   - `THEME_ORDER` array controls display order in the switcher
   - `THEME_STORAGE_KEY` constant + `isThemeId` type guard for localStorage parsing
3. **Created React context** at `src/context/ThemeContext.tsx`:
   - `ThemeProvider` reads stored theme from localStorage on mount
   - `setThemeId` applies new theme by iterating the 19 tokens and calling `document.documentElement.style.setProperty("--{token}", value)` for each
   - Writes `data-theme` and `data-theme-mode` attributes to `<html>` for debug/targeting
   - Hydration guard: `mounted` flag so UI doesn't flash the default name before localStorage is read
4. **Created switcher UI** at `src/components/ui/ThemeSwitcher.tsx`:
   - Fixed bottom-left (`fixed bottom-4 left-4 z-[60] print:hidden`)
   - Collapsed: pill with 4-dot preview + theme name + palette icon
   - Expanded: popover above the pill, 2-col grid of theme chips (~340px wide, max 70vh scroll)
   - Each chip: 4-dot color preview, name, "Dark" badge for soft-dark themes, checkmark overlay when active
   - Click-to-apply (no hover preview, per research), popover stays open for rapid auditioning
   - Keyboard: arrow keys / Home / End navigate chips, Enter/Space applies, Escape closes + restores focus to trigger
   - A11y: `role="radiogroup"` + `role="radio"` + `aria-checked`, `aria-expanded` + `aria-controls` on trigger, focus trap via `rAF`-delayed focus-on-open
   - `motion-reduce:transition-none` everywhere there's a transition
   - Outside-click closes the popover
5. **Wired the provider + switcher** into `src/components/providers/Providers.tsx` (`ThemeProvider` wraps children, `<ThemeSwitcher />` sibling of children).
6. **Added FOUC-prevention inline script** to `src/app/layout.tsx`:
   - Pre-computed theme token map (11 themes Ă— 19 tokens) baked as JSON into the script at build time
   - Script reads `localStorage["iffer-theme"]`, looks up the token map, and applies properties to `document.documentElement` synchronously before React hydrates
   - Wrapped in `try/catch` â€” safe in private mode or localStorage-disabled browsers
   - Also sets `document.documentElement.dataset.theme`
7. **Added theme color transition** to `globals.css`: 200ms `background-color` + `color` transition on `body`, gated by `@media (prefers-reduced-motion: no-preference)`.
8. **Wrote feature doc** at `docs/features/theme-switcher.md` with architecture diagram, how-it-works, palette catalog, and "how to add a theme" guide.
9. **Build verified** â€” `npm run build` passes, all 21 static pages generate, TypeScript clean.
